# Processors & Items Registration

All content is registered as static fields in `setup/ModEntries.java` using the static builder methods inherited from `ModEntryBuilder`.

## ModEntryBuilder API

| Method | Description |
|---|---|
| `add(name)` | Start a blank entry |
| `addItem(name)` | Register a simple item with default properties |
| `addItem(name, supplier)` | Register a custom item |
| `addProcessor(name)` | Register a full processor (block + block entity + menu + energy + recipes) |
| `.block(supplier)` | Set the block supplier |
| `.item(supplier)` | Set a custom item supplier (overrides auto BlockItem) |
| `.blockEntity(constructor)` | Set the block entity constructor |
| `.menu(factory)` | Set the container/menu factory |
| `.withEnergyInput(capacity)` | Define energy storage (input rate = capacity/2) |
| `.withRecipes()` | Register a `RecipeType` and `RecipeSerializer` |
| `.withRecipes(typeSupplier, serializerSupplier)` | Register custom recipe type/serializer |
| `.fluidCap(in, out, default)` | Define fluid capability (input/output/default tanks) |
| `.itemCap(in, out)` | Define item capability (input/output slots) |
| `.material(color)` | Attach a `MaterialEntry` |
| `.build()` | Finalize and register everything, returns a `ModEntry` |

## Register a Full Processing Machine

### 1. Add the entry

```java
// setup/ModEntries.java
public static final ModEntry EXAMPLE_MACHINE = addProcessor("example_machine")
        .fluidCap(1, 1, 0)   // 1 input tank, 1 output tank, 0 extra tanks
        .itemCap(1, 0)        // 1 input slot, 0 output slots
        .build();
```

`addProcessor` is a shortcut that sets:
- `UniversalProcessorBlock` as the block
- `UniversalProcessorBE` as the block entity
- `UniversalProcessorContainer` as the menu
- `100_000 FE` energy storage
- `UniversalProcessorRecipe` type + serializer

The machine is placed in the **Functional Blocks** creative tab. The GUI screen is registered on the client automatically.

### 2. Place block textures

The block is a **horizontal directional** block. Place face textures in a named subfolder:

```
src/main/resources/assets/modtemplate/textures/block/example_machine/
  front.png    - texture for the facing direction
  side.png     - texture for left/right/back faces
  top.png      - (optional) top face
  bottom.png   - (optional) bottom face
```

If only a single `example_machine.png` exists at `textures/block/`, it is used for all faces.

`ModBlockStateProvider` automatically generates the rotated block state and oriented model.

### 3. Run datagen

```bash
./gradlew runData
```

Generates block state JSON with all 4 horizontal rotations, block/item models, loot table, and language entries.

### 4. Add recipes (datagen)

Create a recipe class in `datagen/recipe/` and call `ModRecipeProvider` from it:

```java
// datagen/recipe/ExampleMachineRecipes.java
public class ExampleMachineRecipes {
    public static void generate(RecipeOutput output) {
        processor("example_machine")
                .itemInput(Items.SAND)
                .fluidInput(Fluids.WATER, 1000)
                .fluidOutput(Fluids.LAVA, 1000)
                .processTime(200)
                .energyPerTick(20)
                .save(output, "sand_water_to_lava");
    }
}
```

Register it in `ModRecipeProvider.java`:

```java
ExampleMachineRecipes.generate(output);
```

The recipe builder supports:

| Method | Description |
|---|---|
| `.itemInput(item)` | Single item ingredient |
| `.itemInput(item, count)` | Item ingredient with count |
| `.itemInput(tagKey, count)` | Tag-based ingredient |
| `.fluidInput(fluid, amount)` | Fluid input in mB |
| `.itemOutput(item)` | Single item output |
| `.itemOutput(item, count)` | Item output with count |
| `.fluidOutput(fluid, amount)` | Fluid output in mB |
| `.processTime(ticks)` | Processing duration (default 200) |
| `.energyPerTick(fe)` | Energy cost per tick (default 20) |

Recipe JSONs are generated in `data/modtemplate/recipe/<processor_name>/`.

### 5. Access the entry at runtime

```java
Block block = ModEntries.EXAMPLE_MACHINE.block().get();
MenuType<?> menu = ModEntries.EXAMPLE_MACHINE.menu().get();
RecipeType<?> type = ModEntries.EXAMPLE_MACHINE.recipeType().get();

// or by name
ModEntry entry = ModEntries.get("example_machine");
```

## Recipe JSON Format

Recipes generated by `UniversalProcessorRecipeBuilder` or written by hand follow this format:

```json
{
  "type": "modtemplate:example_machine",
  "item_inputs": [
    { "ingredient": { "item": "minecraft:sand" }, "count": 1 }
  ],
  "fluid_inputs": [
    { "fluid": "minecraft:water", "amount": 1000 }
  ],
  "item_outputs": [],
  "fluid_outputs": [
    { "fluid": "minecraft:lava", "amount": 1000 }
  ],
  "process_time": 200,
  "energy_per_tick": 20
}
```

Place hand-written recipe files in:

```
src/main/resources/data/modtemplate/recipe/example_machine/
  my_recipe.json
```
