# Materials Registration

## Register a Simple Item

### 1. Add the entry

```java
// setup/ModEntries.java
public static final ModEntry EXAMPLE_ITEM = addItem("example_item").build();
```

Registers an `Item` with default properties. Placed in the **Ingredients** creative tab automatically.

To use a custom `Item` subclass, pass a supplier as the second argument:

```java
public static final ModEntry SPECIAL_ITEM = addItem("special_item",
        () -> new MyCustomItem(new Item.Properties().stacksTo(16))
).build();
```

Where `MyCustomItem` extends `Item` and adds any custom behaviour you need.

### 2. Place the item texture

```
src/main/resources/assets/modtemplate/textures/item/
  example_item.png
```

Item model JSON is generated by `ModItemModelProvider` during datagen. If the texture file exists the model resolves correctly; otherwise you will see a missing texture.

### 3. Run datagen

```bash
./gradlew runData
```

Generated output:
- `src/generated/resources/assets/modtemplate/models/item/example_item.json`
- Language entry in `en_us.json`

## Register a Plain Block (no block entity)

### 1. Add the entry

```java
public static final ModEntry DECORATIVE_BLOCK = add("decorative_block")
        .block(() -> new Block(BlockBehaviour.Properties.of().strength(2.0f)))
        .build();
```

Registers the block + auto-generated `BlockItem`. Placed in the **Building Blocks** creative tab.

To use a custom `Block` subclass, swap the supplier:

```java
public static final ModEntry DECORATIVE_BLOCK = add("decorative_block")
        .block(() -> new MyCustomBlock(BlockBehaviour.Properties.of().strength(2.0f)))
        .build();
```

### 2. Place block texture

For a simple cube-all block:

```
src/main/resources/assets/modtemplate/textures/block/
  decorative_block.png
```

`ModBlockStateProvider` detects the single texture and generates a `cube_all` model.

### 3. Run datagen

Generates block state JSON, block/item model JSONs, and a loot table (drops itself).

The material system lets you register a complete metal/mineral material (ore, ingot, dust, plate, nugget, raw ore, storage block, molten fluid, and bucket) with a single builder call. All entries are defined in `setup/ModEntries.java`.

## Quick Start

```java
public static final ModEntry SILVER = addMetalOreMaterial("silver", Color.LIGHT_GRAY.getRGB()).build();
```

This single call registers **all** of the following automatically:

| Component | Registry name |
|---|---|
| Ore block + BlockItem | `silver_ore` |
| Storage block + BlockItem | `silver_block` |
| Ingot item | `silver_ingot` |
| Dust item | `silver_dust` |
| Plate item | `silver_plate` |
| Nugget item | `silver_nugget` |
| Raw ore item | `raw_silver` |
| Molten fluid (source + flowing) | `molten_silver` |
| Liquid block | `molten_silver_fluid_block` |
| Bucket item | `molten_silver_bucket` |

Creative tab placement, block states, item models, language entries, and block/item/fluid tags are all handled by datagen automatically.

## Removing Unwanted Components

Use the fluent `MaterialEntry` API to skip parts you do not need:

```java
public static final ModEntry COPPER_EXTRA = addMetalOreMaterial("copper_extra", 0xB87333)
        .noOre()      // skip ore block
        .noRawOre()   // skip raw_copper_extra item
        .noFluid()    // skip molten fluid + bucket
        .build();
```

Available removal methods:

| Method | What it skips |
|---|---|
| `.noOre()` | Ore block and its BlockItem |
| `.noRawOre()` | Raw ore item |
| `.noBlock()` | Storage block and its BlockItem |
| `.noIngot()` | Ingot item |
| `.noDust()` | Dust item |
| `.noPlate()` | Plate item |
| `.noNugget()` | Nugget item |
| `.noFluid()` | Molten fluid, liquid block, and bucket |
| `.worldgenConfig(min, max, 0)` | Disable ore world generation (set qty to 0) |

## Step-by-Step Implementation

### 1. Add the entry in `ModEntries.java`

```java
// src/main/java/igentuman/modtemplate/setup/ModEntries.java
public static final ModEntry SILVER = addMetalOreMaterial("silver", 0xC0C0C0).build();
```

### 2. Place block textures

The block state datagen (`ModBlockStateProvider`) uses `cubeAll` for material blocks, looking for textures in:

```
src/main/resources/assets/modtemplate/textures/block/material/
  silver_ore.png
  silver_block.png
```

If no texture file exists, datagen still generates the model JSON but the texture will be missing in-game (purple/black).

### 3. Place item textures

Item textures use per-type subfolders under `item/material/`. The filename is the **material name** (not the full item registry name):

```
src/main/resources/assets/modtemplate/textures/item/material/ingot/
  silver.png
src/main/resources/assets/modtemplate/textures/item/material/dust/
  silver.png
src/main/resources/assets/modtemplate/textures/item/material/plate/
  silver.png
src/main/resources/assets/modtemplate/textures/item/material/nugget/
  silver.png
src/main/resources/assets/modtemplate/textures/item/material/raw/
  silver.png
src/main/resources/assets/modtemplate/textures/item/material/gem/
  silver.png
```

Bucket item textures are handled by NeoForge's fluid bucket system automatically if you provide the fluid texture.

### 4. Place fluid textures

Molten fluid textures must be placed in:

```
src/main/resources/assets/modtemplate/textures/fluid/
  molten_silver_still.png
  molten_silver_flow.png
```

The `MaterialFluidType` reads these paths from the `FluidDefinition` attached to the material.

### 5. Run datagen

```bash
./gradlew runData
```

This generates:
- `src/generated/resources/assets/modtemplate/blockstates/silver_ore.json`
- `src/generated/resources/assets/modtemplate/blockstates/silver_block.json`
- `src/generated/resources/assets/modtemplate/models/item/silver_ingot.json` (and all other item models)
- `src/generated/resources/assets/modtemplate/lang/en_us.json` entries
- Tag entries in `data/modtemplate/tags/` and `data/c/tags/`

### 6. Add recipes (datagen)

Add your material recipes in `datagen/recipe/`. Create a new class or add to an existing one:

```java
// src/main/java/igentuman/modtemplate/datagen/recipe/SilverRecipes.java
public class SilverRecipes {
    public static void generate(RecipeOutput output) {
        // Ingot from nuggets (shapeless)
        ShapelessRecipeBuilder.shapeless(RecipeCategory.MISC,
                ModEntries.SILVER.materialEntry().ingot().get(), 1)
                .requires(ModEntries.SILVER.materialEntry().nugget().get(), 9)
                .unlockedBy("has_nugget", ...)
                .save(output, "silver_ingot_from_nuggets");

        // Nuggets from ingot
        ShapelessRecipeBuilder.shapeless(RecipeCategory.MISC,
                ModEntries.SILVER.materialEntry().nugget().get(), 9)
                .requires(ModEntries.SILVER.materialEntry().ingot().get())
                .save(output, "silver_nuggets_from_ingot");
    }
}
```

Register it in `ModRecipeProvider`:

```java
// datagen/recipe/ModRecipeProvider.java
SilverRecipes.generate(output);
```

### 7. World generation

<a name="world-generation"></a>

Ore world generation is **automatic** for materials registered via `metalOre()`. `crystalOre()` does not set worldgen defaults - call `.worldgenConfig(...)` explicitly if you need it. No manual JSON or datapack class edits needed.

**Default values** applied by `metalOre()`:

| Parameter | Default |
|---|---|
| Min height | -64 |
| Max height | 64 |
| Vein size | 9 |
| Veins per chunk | 4 |

To override defaults, chain `.worldgenConfig(minHeight, maxHeight, veinSize)` on the `MaterialEntry`:

```java
// ModEntries.java - inside addMetalOreMaterial builder chain
public static final ModEntry SILVER = addMetalOreMaterial("silver", 0xC0C0C0)
        .worldgenConfig(-32, 48, 7)   // minHeight, maxHeight, veinSize
        .build();
```

All three values are also **runtime-configurable** via a generated TOML config at `config/<modid>-worldgen-server.toml`:

```toml
[silver]
  # Minimum Y level for ore generation
  min_height = -32
  # Maximum Y level for ore generation
  max_height = 48
  # Number of blocks per ore vein
  vein_size = 7
  # Number of ore veins attempted per chunk
  veins_per_chunk = 4
```

**What the system auto-registers per material** (when `hasWorldgenConfig()` is true):

- `ModConfiguredFeatures` - ore feature targeting both `STONE_ORE_REPLACEABLES` and `DEEPSLATE_ORE_REPLACEABLES` tag blocks
- `ModPlacedFeatures` - placement with `CountPlacement`, `InSquarePlacement`, `ConfigurableOrePlacement` (reads runtime config for Y range), and `BiomeFilter`
- `ModBiomeModifiers` - adds the placed feature to all `IS_OVERWORLD` biomes at `UNDERGROUND_ORES` generation step

Run `./gradlew runData` to emit the datapack JSONs into `src/generated/resources/data/`.

To **disable worldgen** for a material, pass `qty = 0` to `.worldgenConfig()` or call `.noOre()` - worldgen is skipped automatically when the material has no ore block or `worldgenQty == 0`.

## Accessing Material Items/Blocks at Runtime

```java
ModEntry silver = ModEntries.SILVER;
MaterialEntry mat = (MaterialEntry) silver.materialEntry();

Item ingot   = mat.ingot().get();
Item dust    = mat.dust().get();
Block ore    = mat.oreBlock().get();
Block block  = mat.storageBlock().get();
Fluid fluid  = mat.fluidDefinition().source().get();  // if fluid registered
```
